Quick Start

In this quick tutorial, you'll learn how to create, emulate and deploy your very own first Warped module.

1. Create a directory for the project

Create an empty project using the following commands:

mkdir warping
cd warping

2. Create an empty module

First we will create a simple Node.js module in mymodule ready to be warped:

mkdir mymodule  
cd mymodule  
npm init -y

3. Create index.js module script

There should now be a package.json file inside your project directory. We need to write our code to do something in the module.

A warped module must export asynchronous functions that will be transformed as microservice api calls. In this example, we are simply writing a hello function that returns some text, with the Node.js version.

Create a file and name it index.js in the project directory. Copy and paste the following code to index.js:

exports.hello = function (name) {
  return `Hello ${name} from Node.js ${process.version}`
}

4. Install WarpJS SDK and login

To install the SDK you need Node.js installed on your computer. Look to https://nodejs.org/en/download/ to install it.

Install WarpJS SDK that will provide all the commands to warp our module

npm install warp@dev --save-dev

Then, login using your credentials:

npx warp login

Note that if you don't have a ScaleDynamics account yet, you can ask here https://console.scaledynamics.com/auth/signup/.

5. Warp the module and run emulator

Use this line to create the ../myWarpedModule.js that will be used by our Node.js script.

npx warp build --output umd:../myWarpedModule.js
npx warp emulator run

As warp emulator is running, use a new terminal fo following steps.

6. Create a simple browser script that will call the warped module

First change directory to the warping one we created at the beginning:

cd warping

Now you're back in warping directory create an index.html file containing these lines:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width,initial-scale=1" />
    <title>Warped module use</title>
  </head>
  <body>
    <script src="./myWarpedModule.js"></script>
    <script>
      const MyModule = window['myWarpedModule-client']
      MyModule.loadEngine().then(() => {
        const { hello } = new MyModule()  
        hello('Kirk').then((message) => {  
          document.body.innerHTML += message
        })  
      })
    </script>
  </body>
</html>

Open that file on a local server with the following command, and you will see it works!

npx serve

We're now ready to deploy the module in the cloud.

7. Deploy the microservice in a cloud environment

Get back in the other terminal and CTRL-C the emulator command.

First create a production build

npx warp build --output umd:../myWarpedModule.js

Then deploy the module in a cloud environment.

npx warp deploy

During the deploy,

  • you'll be asked to select a project. Select the project name you want.
  • you'll be asked to select an environment. Create one with the name you want.

By default a cloud environment will use Azure cloud.

When the deployment is done, get back in the browser and reload the page. Your warped module is now running in the cloud.
