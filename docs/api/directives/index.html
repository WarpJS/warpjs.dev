Directives

Directives are used during the build of the frontend and server JavaScript files to indicate the visibility of a function.

A WarpJS directive starts with 'warp',

To indicate the visibility of the code in the frontend asset use -client to indicate that the code will be deleted from the original JavaScript file, or +client to indicate to keep the code in your browser asset.

In the same way, use -server to indicate that the code will be deleted from the server code, or +server to get the code on the server.

Example, indicate the function hello should exist on the server, but not on the frontend:

// warp function
const hello = (name) => {
  // warp directive
  'warp +server -client'

}
