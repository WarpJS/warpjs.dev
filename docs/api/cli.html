WarpJS Command-line Interface

Install the CLI

To install the CLI, open your terminal in a Node.js project and install with npm:

$ npm install warp

Listing commands

To get a list of commands, run:

$ npx warp help
$ npx warp -h

To get details about a command's options, run:

$ npx warp help [command]
$ npx warp <command> -h

Available commands

build

Generate the client:

$ npx warp build [options] <project-dir>

Options can be omitted if you use warp.config.js.

The CLI overrides values from warp.config.js.

Arguments

ArgumentDescription
<project-dir>the path to your server project

Options

OptionDescription
-o, --output <format>:<path>format (umd or node-module) and path of output (overrides values from warp.config.js)

All relative paths from the CLI are relative to PWD.

Output Formats

WarpJS supports several formats to generate the JavaScript client:

umd

Universal Module Definition (UMD) is capable of working everywhere, be it in the client, on the server or elsewhere.

It can be useful to import the client with an HTML <script> tag.

$ npx warp build -o umd:./client/src/warp-client.js ./server
node-module

Use it to generate a npm module with its package.json.

The path has to be the target directory containing the node_modules directory.

$ npx warp build -o node-module:./client/ ./server

Examples

$ npx warp build -o umd:./client/src/warp-client.js ./server
$ npx warp build -o node-module:./client/ ./server

# With a warp.config.js file in server project
$ npx warp build ./server

deploy

Deploy the serverless functions and assets of your project:

$ npx warp deploy [options] <project-dir>

Arguments

ArgumentDescription
<project-dir>the path to your server project

Options

OptionDescription
--asset-dir <asset-directory>the path to your static front end if you want to deploy it in addition of serverless functions

All relative paths from the CLI are relative to PWD.

Examples

$ npx warp deploy .
$ npx warp deploy --asset-dir ../build .

login

Log into your WarpJS account:

$ npx warp login [options] [e-mail]

Create an account on your WarpJS console. WarpJS is currently being tested by a wide community of developers. You need to get a link from a member with an active account. If you don't know any member, please request a Private Beta Access on warpjs.com by clicking on the Join Free button.

Options

OptionDescription
-p, --password <password>password

Warning Useful if you want to use a CI/CD but be careful, the password can be read in plain text in different systems.

Examples

$ npx warp login
$ npx warp login email@domain.com
$ npx warp login email@domain.com -p my-password

logout

Log out of your WarpJS account:

$ npx warp logout

start-emulator

Launch the emulator:

$ npx warp start-emulator [options] <project-dir>

Options can be omitted if you use warp.config.js.

The CLI overrides values from warp.config.js.

Arguments

ArgumentDescription
<project-dir>the path to your server project

Options

OptionDescription
-o, --output <format>:<path>format (umd or node-module) and path of output (overrides values from warp.config.js)
-p, --port <number>use custom port number (default: 3192 or value from warp.config.js)
-w, --watchenable live-reload mode that watch the project files and rebuild the project whenever they change

All relative paths from the CLI are relative to PWD.

Output Formats

WarpJS supports several formats to generate the JavaScript client:

umd

Universal Module Definition (UMD) is capable of working everywhere, be it in the client, on the server or elsewhere.

It can be useful to import the client with an HTML <script> tag.

$ npx warp start-emulator -o umd:./client/src/warp-client.js ./server -w

Options can be omitted if you use warp.config.js.

The CLI overrides values from warp.config.js.

node-module

Use it to generate a npm module with its package.json.

The path has to be the target directory containing the node_modules directory.

$ npx warp start-emulator -o node-module:./client/ ./server -w

Examples

$ npx warp start-emulator -o umd:./client/src/warp-client.js ./server -w
$ npx warp start-emulator -o node-module:./client/ ./server -w

stop-emulator

Stop the emulator:

$ npx warp start-emulator [options] <project-dir>

Arguments

ArgumentDescription
<project-dir>the path to your server project

Options

OptionDescription
-p, --port <number>use custom port number (default: 3192 or value from warp.config.js)

Examples

$ npx warp stop-emulator ./server
$ npx warp stop-emulator -p 3191 ./server

version

Print version of WarpJS:

$ npx warp version
