WarpJS configuration file

The WarpJS configuration file is optional, the CLI being sufficient but you can configure easily some parameters in the server project. The CLI overrides values from warp.config.js.

Create a warp.config.js at the root of the server project.

Example

A WarpJS configuration file is a module that exports a default object with options:

module.exports = {
  project: "project-name",
  output: {
    format: "umd",
    filePath: "../client/src/functions.js",
  },
  emulator: {
    port: 3192,
    loggingLevel: "info",
  },
};

All relative paths from the warp.config.js are relative to this file.

Attribute List

project

The name of project used to create the serverless project.

You have to create a project on your WarpJS console. Sign in and add a project with the same name as your configuration.

By default, WarpJS uses the package.json.name attribute but it can be changed by this attribute.

Rules

A project name must be less than or equal to 214 characters and must only contain [a-z], [0-9], underscore (_), dash (-), dot (.).

output

Used by warp build and warp start-emulator commands to generate the JavaScript client output.

WarpJS supports several formats to generate the JavaScript client:

umd

Universal Module Definition (UMD) is capable of working everywhere, be it in the client, on the server or elsewhere.

It can be useful to import the client with an HTML <script> tag.

output: {
  format: "umd",
  filePath: "../client/src/functions.js",
},
node-module

Use it to generate a npm module with its package.json.

The path has to be the target directory containing the node_modules directory.

output: {
  format: "node-module",
  projectPath: "../client",
},

emulator

The WarpJS emulator starts with the warp start-emulator command, and stops with warp stop-emulator

emulator: {
  port: 3192,
  loggingLevel: "info",
}

port

WarpJS emulator port number.

Default: 3192

loggingLevel

Available log levels: none, error, warn, info.

Default: info
