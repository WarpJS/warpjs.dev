Quick Start

Requirements

Before starting, make sure you meet the following requirements:

  • Install Node.js latest 8.x or above (which can be checked by running node -v)

  • Get your User ID key from your WarpJS account (sign in or sign up to create an account)

  • WarpJS is available for Linux & macOS (Windows coming soon!)

Setup

In this tutorial, we introduce the different steps to initialize a project with WarpJS, then create a Warp function, run locally and finally deploy it on a serverless infrastructure effortlessly!

Note that to skip the tutorial to go quickly, you can git clone the full project samples/hello-world, set your userId and run it.

Init Project

To take advantage of the WarpJS technology, you will need to build your JavaScript project using Webpack, one of the most popular bundlers. To quick start, you can use our starter template that contains this prerequisite and the most common webpack plugins to give you a great developer experience.

So open your terminal and let's create a new project hello-world from our webpack starter template:

npx degit warpjs/webpack-template hello-world
cd hello-world
npm install

If you have any trouble with node-gyp after install, please see the installation doc.

You will get the following project structure:

 hello-world
  |- package.json
  |- package-lock.json
  |- webpack.config.js
  |- /public
    |- favicon.ico
    |- logo.svg
    |- styles.css
  |- /src
    |- index.html
    |- index.js

Install WarpJS Dependencies

First of all you have to install with npm the core libraries required to run WarpJS:

npm install --save @warpjs/warp @warpjs/engine
npm install --save-dev @warpjs/webpack-plugin

Update Webpack Config

Update your webpack.config.js by adding the WarpJS Webpack Plugin.

First, import the plugin at the top of the file:

const WarpifyPlugin = require('@warpjs/webpack-plugin')

Then, add the following plugin configuration:

module.exports = {
  plugins: [
    //...
    new WarpifyPlugin({
      exclude: [/node_modules/],
      include: [/\.js$/],
      config: {
        project: {
          userId: 'YOUR_USER_ID', // insert your "User ID" here
          name: 'demo'
        }
      }
    })
  ]
}

And replace the userId value by your own (get it from the WarpJS Console).

Boostrap WarpJS

In your src/index.js file, you must first import the WarpJS packages:

import { defaultWarper as warper } from '@warpjs/warp'
import engine from '@warpjs/engine'

Then, initialize the WarpJS engine, without options as in the following example:

engine.init()

Advanced options can be given as parameter to the init(options) method.
See the documentation about @warpjs/engine.

Now you are going to create your first Node.js function directly in your front-end source code.
The following function hello receives a name as parameter, and returns a custom greeting message with the name and the current Node.js version from the back-end runtime:

const hello = (name) => {
  return `Hello ${name} from Node.js ${process.version} 👋`
}

Then you have to add the WarpJS directive to enable the WarpJS processing on your function, and allow it to be run serverless:

const hello = (name) => {
  'warp +server -client'

  return `Hello ${name} from Node.js ${process.version} 👋`
}

You can learn more about warp directives in the api/directives section.

Finally, on the loading page, you will call that hello function from a WarpJS warper warper.call(...), and handle the response as an async Promise:

warper.call(hello, 'World').then(response => {
  result.innerHTML = response
})

In the end, your complete JavaScript file should look like the following snippet:

src/index.js

// import WarpJS
import { defaultWarper as warper } from '@warpjs/warp'
import engine from '@warpjs/engine'

// init WarpJS
engine.init()

// warp function
const hello = (name) => {
  // warp directive
  'warp +server -client'

  return `Hello ${name} from Node.js ${process.version} 👋`
}

// on load, call function with WarpJS
warper.call(hello, 'World').then(response => {
  result.innerHTML += response
})

Update npm Scripts

You can set up a few little shortcuts to help you in the development environment.
Let's adjust your package.json by adding the following npm scripts:

package.json

  "scripts": {
    "start": "webpack-dev-server --open",
    "build": "webpack --progress --mode=production",
    "deploy": "warp-deploy --asset-dir=dist ."
  },

Basic Usage

Your project is now ready to run & deploy 🚀

Development

First, try to run it on development mode from your local environment:

# build & run a dev server
npm start

The development mode has a live reload capability on every code change.

Production

Finally, you can deploy your WarpJS function in production to serverless infrastructure, simply with the following commands:

# build app
npm run build

# deploy to prod
npm run deploy

Then the deploy script will return the public application URL.

Now you can manage your project from your WarpJS Console.

Conclusion

In this tutorial, you followed the different steps to initialize a project with WarpJS, then create a Warp function, run and deploy it on a serverless infrastructure effortlessly!

Resources
