Quick Start

In this tutorial, we will initialize a web project with WarpJS and deploy it as a public web site on a serverless infrastructure effortlessly!

Requirements

Before starting, make sure you meet the following requirements:

  • Install Node.js latest 8.x or above (which can be checked by running node -v)

  • Sign into your WarpJS account (or sign up to create an account)

Initialize your project

Create a new project from our WarpJS Starter Template from your terminal:

git clone https://github.com/WarpJS/warpjs-starter-template.git
cd warpjs-starter-template
npm install

If you don't have Git, download the starter template from the zip archive.

If you have any trouble with node-gyp after install, see the official "node-gyp" installation doc.

Login to WarpJS

Run the warp login command to allow to authenticate to your WarpJS account on next steps:

npx warp login

Create a Warp function

WarpJS lets you create your backend functions with no HTTP call to fully automate the deployment to an infrastructure. In your front-end code, you just warp call your JavaScript functions as back-end endpoint.

In the current project, you will call the back-end hello() function through the warp.call(...) helper, in order to return the Node.js version on the page load.

📄 src/index.js

// ...

// back-end warp function
const hello = (name) => {
  'warp +server -client'

  return `Hello ${name} from Node.js ${process.version}`
}

// on web page load
window.addEventListener('DOMContentLoaded', async () => {
  // call back-end function
  const response = await warp.call(hello, 'World')
  // show result in browser
  document.getElementById('result').innerHTML = response
})

Deploy your site

Your project is now ready to deploy & run on production 🚀

Run simply the deploy command, to deploy both your WarpJS functions on a serverless platform and your public assets (html, js, css, ...) on a CDN:

npm run deploy

After execution, the deploy script will return the public URL of your web app.

Then, you can manage your project from the WarpJS Console.

Development tools

Next, you can test locally your web site on a serverless development environment with the dev command:

npm run dev

The development mode starts a serverless local emulator to run your WarpJS functions, and enables a live reload on every code change, for both front-end and back-end.

Feel free to modify the hello function and re-deploy it to feel the developer experience, from coding to deployment.
