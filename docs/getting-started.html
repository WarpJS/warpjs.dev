Quick Start

In this tutorial, we will initialize a web project with WarpJS Serverless and deploy it as a public web site on a serverless infrastructure effortlessly!

Requirements

Before starting, make sure you meet the following requirements:

  • Install Node.js latest 8.x or above (which can be checked by running node -v)

  • Sign into your WarpJS account (or sign up to create an account)

Login to WarpJS

Run the warp login command to allow authenticating to your WarpJS account on next steps:

$ npx warp login

Initialize your project

Create a new project from our WarpJS Starter Template from your terminal:

$ git clone https://github.com/WarpJS/warpjs-starter-template.git
$ cd warpjs-starter-template
$ npm install

If you don't have Git, download the starter template from the zip archive.

If you have any trouble with node-gyp after install, see the official "node-gyp" installation doc.

Create a Warp function

WarpJS allows you to quickly create your back-end functions with no HTTP call, to fully automate the deployment to an infrastructure, and easily call them as simple JavaScript functions.

To start, let's create a new back-end function that will act as web proxy to call a public REST API through the popular axios module.

First, install axios with npm:

$ npm install axios

Create the fetchData back-end function in your src/index.js file:

📄 src/index.js

const axios = require("axios");

const fetchData = async () => {
  const { data } = await axios.get("https://jsonplaceholder.typicode.com/users");
  return data;
};

module.exports = { fetchData };

Build a client

In order to call your back end from any JavaScript project, you have to generate a client with the build command:

$ npx warp build --output umd:dist/client.js .

On this example, you will create a dist/client.js file on the usual UMD module format.

Deploy your project

Your project is now ready to deploy & run on production 🚀

Just launch the deploy command to deploy both your WarpJS functions on a serverless platform and your front end on a CDN:

$ npx warp deploy --asset-dir dist/ .

After execution, the deploy script will return the public URL of your project.

Now you can manage all your projects from the WarpJS Console.

Note that your project will be deployed with a identifier name defined in the warp.config.js, or fallback to the name property of your package.json. By default, this value is set to "demo".

Call function from client

Your WarpJS Serverless function is now ready to be call from any JavaScript project.

In a new client project, first install the @warpjs/engine module with npm:

$ npm add @warpjs/engine

The engine must be imported only once at the application initialization.

So import the engine first, then import your generated client as module, initialize it and call your back-end function as a promise:

import "@warpjs/engine";
import Warp from "./client.js";

const { fetchData } = new Warp();

fetchData().then((data) => {
  console.log(data);
});

Or through <script> tags in your HTML file, like in the dist/index.html file:

<script src="https://cdn.warpjs.com/warp-engine/3.0.7/engine-web-prod.js"></script>
<script src="./client.js"></script>
<script>
  const Warp = window["demo-client"];
  const { fetchData } = new Warp();
  fetchData().then(...);
</script>

Development tools

Next, you can test locally your back end on a serverless development environment with the start-emulator command:

$ npx warp start-emulator --output umd:dist/client.js --watch .

The development mode starts a serverless local emulator to run your WarpJS functions, and enables a live reload on every code change on back-end side.

Then open a new terminal to start a web server, run your generated client in your browser and see the API response live:

$ npx http-server -a 127.0.0.1 -o -c-1 dist/

Feel free to modify the fetchData function and re-deploy it to feel the developer experience, from coding to deployment.
