Basic Setup

From the WarpJS Starter Template used on the Quick Start, we will details each step to setup a project with WarpJS, create a "hello world" back-end function, test locally and finally deploy it on production.

Configure webpack

To take advantage of the WarpJS technology, you have to build your JavaScript project using Webpack, one of the most popular bundlers.

Our starter project includes webpack and the most common webpack plugins to give you a great developer experience. If you prefer start a new project from scratch, see the documentation about advanced setup.

Install Warp plugin

Install with npm the @warpjs/webpack-plugin

npm install --save-dev @warpjs/webpack-plugin

Please note that the WarpJS webpack plugin is only compatible with webpack 4.

Update webpack config

  1. Open your webpack.config.js file.

  2. Import the plugin package:

const WarpifyPlugin = require('@warpjs/webpack-plugin')
  1. Create a new plugin instance:
module.exports = {
  plugins: [
    //...
    new WarpifyPlugin()
  ]
}

By default, your WarpifyPlugin is configured to use the "demo" project from your WarpJS account.
You can create a new project on your WarpJS console, and then configure it in the webpack plugin as follows:

new WarpifyPlugin({
  config: {
    project: {
      name: "YOUR_PROJECT_NAME"
    }
  }
}

Create a Warp function

Note that the project will use a Node 8 runtime with ES2017 syntax.

Install packages

Install with npm the core libraries required to run WarpJS:

npm install --save @warpjs/engine
npm install --save @warpjs/warp

Init engine

Initialize the WarpJS engine once at application start:

import engine from '@warpjs/engine'

engine.init()

See the documentation about @warpjs/engine.

Add directive

Next, you have to add the WarpJS directive (eg. 'warp +server -client') to indicate WarpJS to manage your JavaScript function, and allow it to be run serverless:

const hello = (name) => {
  'warp +server -client'

  return `Hello ${name} from Node.js ${process.version} 👋`
}

See the documentation about directives.

Warp call

Finally, you have to call the warp function from the warp.call(...) helper, and handle the response as an async Promise:

import { defaultWarper as warp } from '@warpjs/warp'

warp.call(hello, 'World').then(response => {
  result.innerHTML = response
})

See the documentation about warp.call().

Update npm scripts

Now you can set up a few little shortcuts to help you in the development environment.

Add the following npm scripts in your package.json file:

  "scripts": {
    "dev": "webpack-dev-server --open",
    "build": "webpack --progress --mode=production",
    "predeploy": "npm run build",
    "deploy": "warp deploy --asset-dir=dist ."
  },

Basic Usage

Development

Run your project on development mode from your local environment:

# run a dev server
npm run dev

Production

Deploy your WarpJS project in production with the following commands:

# build & deploy to prod
npm run deploy
